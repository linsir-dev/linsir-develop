# Java 关键字示例

本目录包含 Java 关键字的示例代码，展示了 `final`、`finalize` 和 `finally` 的使用方法和区别。

## 目录结构
```
keywords/
├── FinalDemo.java         // final 关键字示例
├── FinalizeDemo.java      // finalize 方法示例
└── FinallyDemo.java       // finally 代码块示例
```

## 示例代码功能

### 1. final 关键字 (`FinalDemo.java`)
- **修饰变量**：
  - 一旦赋值，不可更改
  - 对于引用类型，引用不可变，但对象内容可变
  - 成员变量必须初始化（在声明时或构造器中）
- **修饰方法**：
  - 不可被子类重写
  - 可以被继承和调用
- **修饰类**：
  - 不可被继承
  - 如 `String` 类就是 final 类

### 2. finalize 方法 (`FinalizeDemo.java`)
- **功能**：
  - Object 类的方法，在垃圾回收器 (GC) 准备释放对象内存之前调用
  - 已在 JDK 9 中被标记为 Deprecated (过时)，不推荐使用
- **缺点**：
  - 执行时间不确定
  - 性能差
  - 可能导致对象复活
- **示例**：
  - 演示了对象复活的概念
  - 展示了 finalize 方法只会被调用一次的特性

### 3. finally 代码块 (`FinallyDemo.java`)
- **功能**：
  - 无论是否发生异常，finally 代码块总会执行（除非 System.exit()）
  - 在 try 或 catch 中有 return 时，finally 依然会执行
- **使用场景**：
  - 资源释放（如关闭文件、数据库连接等）
  - 清理操作
- **注意事项**：
  - 不建议在 finally 中使用 return，这会覆盖 try/catch 中的返回值

## 关键字区别总结
| 关键字/方法 | 作用 | 使用场景 | 注意事项 |
|------------|------|----------|----------|
| final | 修饰变量、方法、类 | 不可变的常量、不可重写的方法、不可继承的类 | 引用类型的对象内容可变 |
| finalize | 对象被回收前调用的方法 | 资源释放（已过时） | 执行时机不确定，可能导致对象复活 |
| finally | 异常处理的代码块 | 资源释放、清理操作 | 总会执行，除非 System.exit() |
