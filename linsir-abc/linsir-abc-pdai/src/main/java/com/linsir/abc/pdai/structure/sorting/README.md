# 排序算法示例代码

## 1. 项目结构

本目录包含常见排序算法的示例代码，包括：

- 比较排序算法：
  - BubbleSort.java - 冒泡排序
  - SelectionSort.java - 选择排序
  - InsertionSort.java - 插入排序
  - MergeSort.java - 归并排序
  - QuickSort.java - 快速排序
  - HeapSort.java - 堆排序
  - ShellSort.java - 希尔排序

- 非比较排序算法：
  - CountingSort.java - 计数排序
  - BucketSort.java - 桶排序
  - RadixSort.java - 基数排序

- 测试类：
  - SortingTest.java - 排序算法测试类

## 2. 排序算法性能比较

| 排序算法 | 平均时间复杂度 | 最坏时间复杂度 | 最好时间复杂度 | 空间复杂度 | 稳定性 |
|---------|---------------|---------------|---------------|-----------|-------|
| 冒泡排序 | O(n²)         | O(n²)         | O(n)          | O(1)      | 稳定 |
| 选择排序 | O(n²)         | O(n²)         | O(n²)         | O(1)      | 不稳定 |
| 插入排序 | O(n²)         | O(n²)         | O(n)          | O(1)      | 稳定 |
| 归并排序 | O(n log n)    | O(n log n)    | O(n log n)    | O(n)      | 稳定 |
| 快速排序 | O(n log n)    | O(n²)         | O(n log n)    | O(log n)  | 不稳定 |
| 堆排序   | O(n log n)    | O(n log n)    | O(n log n)    | O(1)      | 不稳定 |
| 希尔排序 | O(n^1.3)      | O(n²)         | O(n)          | O(1)      | 不稳定 |
| 计数排序 | O(n + k)      | O(n + k)      | O(n + k)      | O(k)      | 稳定 |
| 桶排序   | O(n + k)      | O(n²)         | O(n)          | O(n + k)  | 稳定 |
| 基数排序 | O(n * k)      | O(n * k)      | O(n * k)      | O(n + k)  | 稳定 |

注：k 表示数据范围或位数

## 3. 排序算法适用场景

| 排序算法 | 适用场景 |
|---------|---------|
| 冒泡排序 | 数据量小，基本有序 |
| 选择排序 | 数据量小，对稳定性无要求 |
| 插入排序 | 数据量小，基本有序 |
| 归并排序 | 数据量大，要求稳定 |
| 快速排序 | 数据量大，对稳定性无要求 |
| 堆排序   | 数据量大，对稳定性无要求 |
| 希尔排序 | 数据量中等，对稳定性无要求 |
| 计数排序 | 数据范围小，整数排序 |
| 桶排序   | 数据均匀分布，范围较大 |
| 基数排序 | 整数或字符串排序，位数较少 |

## 4. 运行测试

### 4.1 编译和运行

在项目根目录执行以下命令：

```bash
# 编译项目
mvn compile

# 运行测试
mvn exec:java -Dexec.mainClass="com.linsir.abc.pdai.structure.sorting.SortingTest"
```

### 4.2 测试结果

测试将输出每种排序算法的排序前后结果，包括：
- 随机数组
- 已排序数组
- 逆序数组

## 5. 代码特点

1. **完整实现**：每种排序算法都有完整的实现，包括核心排序逻辑和测试方法
2. **详细注释**：代码中包含详细的注释，解释算法原理和实现细节
3. **易于理解**：代码结构清晰，变量命名规范，便于学习和理解
4. **性能考虑**：在实现中考虑了算法的性能优化，如冒泡排序的提前退出
5. **通用性**：算法实现适用于各种输入情况，包括空数组、单元素数组等边界情况

## 6. 学习建议

1. **从简单到复杂**：先学习冒泡排序、选择排序、插入排序等简单算法
2. **理解原理**：掌握每种算法的基本原理和时间复杂度分析
3. **比较学习**：对比不同算法的优缺点，理解适用场景
4. **实践应用**：尝试在实际项目中应用合适的排序算法
5. **优化思考**：思考如何进一步优化算法性能
